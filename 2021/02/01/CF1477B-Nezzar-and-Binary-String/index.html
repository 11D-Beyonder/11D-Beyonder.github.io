CF1477B Nezzar and Binary String

发表于2021-02-01|更新于2023-01-31|题解
|字数总计:1.5k|阅读时长:7分钟|阅读量:

Nezzar has a binary string sss of length nnn that he wants to share with his best friend, Nanako. Nanako will spend qqq days inspecting the binary string. At the same time, Nezzar wants to change the string sss into string fff during these qqq days, because it looks better.
It is known that Nanako loves consistency so much. On the iii-th day, Nanako will inspect a segment of string sss from position lil_ili​ to position rir_iri​ inclusive. If the segment contains both characters ‘0’ and ‘1’, Nanako becomes unhappy and throws away the string.
After this inspection, at the iii-th night, Nezzar can secretly change strictly less than half of the characters in the segment from lil_ili​ to rir_iri​ inclusive, otherwise the change will be too obvious.
Now Nezzar wonders, if it is possible to avoid Nanako being unhappy and at the same time have the string become equal to the string fff at the end of these qqq days and nights.

Input

The first line contains a single integer ttt (1≤t≤2⋅1051≤t≤2⋅10^51≤t≤2⋅105) — the number of test cases.
The first line of each test case contains two integers nnn ,qqq (1≤n≤2⋅105, 0≤q≤2⋅105).
The second line of each test case contains a binary string sss of length nnn.
The third line of each test case contains a binary string fff of length nnn.
Then qqq lines follow, iii-th of them contains two integers lil_ili​, rir_iri​ (1≤li≤ri≤n1≤l_i≤r_i≤n1≤li​≤ri​≤n) — bounds of the segment, that Nanako will inspect on the iii-th day.
It is guaranteed that the sum of nnn for all test cases doesn’t exceed 2⋅1052⋅10^52⋅105, and the sum of qqq for all test cases doesn’t exceed 2⋅1052⋅10^52⋅105.

Output

For each test case, print “YES” on the single line if it is possible to avoid Nanako being unhappy and have the string fff at the end of qqq days and nights. Otherwise, print “NO”.
You can print each letter in any case (upper or lower).

Note

In the first test case, 00000‾→000‾11→00111\underline{00000} \rightarrow \underline{000}11 \rightarrow 0011100000​→000​11→00111 is one of the possible sequences of string changes.
In the second test case, it can be shown that it is impossible to have the string fff at the end.

Translation

给出两个二进制字符串 sss 和 fff，有 qqq 次操作，目标是将 sss 变为 fff。每次操作给一个区间 [li,ri][l_i,r_i][li​,ri​]，若此时 sli∼sris_{l_i}\sim s_{r_i}sli​​∼sri​​ 中 000 和 111 都存在，就定义为失败；否则可以修改严格小于区间一半长度数量的字符，最后若能达到目标就成功。

Idea

正向考虑非常麻烦，因为要顾及在后面的操作中不能让一段区间中出现既有 000 又有 111 的情况；不妨反向思考，一次反向考虑 qqq 次操作，从 [lq,rq][l_q,r_q][lq​,rq​] 到 [l1,r1][l_1,r_1][l1​,r1​]，将 fff 变为 sss。
我们首先考虑 fff 一定能变为 sss，那么第 qqq 次操作后，sss 就最终变成了 fff。考察 fff 中 [lq,rq][l_q,r_q][lq​,rq​] 这段区间，我们必须要保证在这次操作前 [lq,rq][l_q,r_q][lq​,rq​] 这段区间全 000 或全 111，然后将这段区间修改为 flq∼frqf_{l_q}\sim f_{r_q}flq​​∼frq​​。逆向操作，如果区间内有严格多数的 000 或 111，将区间内数字都变为多数，最后比较最后得到的字符串是否为 sss，这种方法带有反证法的思想。
用到的数据结构是区间修改、单点查询的线段树；叶节点是该位置上的数字（000 或 111），结点维护区间和，即区间内 111 的个数。
