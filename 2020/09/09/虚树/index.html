虚树

发表于2020-09-09|更新于2023-01-31|数据结构
|字数总计:2k|阅读时长:8分钟|阅读量:

引入

进行树形DP时，往往要逐个遍历树上的点进行状态转移；若树有 nnn 个节点，那么时间复杂度至少是 O(n)O(n)O(n)。
假设对某一问题进行 mmm 次询问，由于每一次都需要遍历整棵树进行树形DP，所以时间复杂度至少为 O(mn)O(mn)O(mn)。在询问次数较多的情况下，容易 TLE\text{TLE}TLE。
在处理某些问题时，原树上的一些信息是不必要的，只需要访问部分节点就可得到答案；也就是说，应当把一部分无用的节点舍去，不予访问。因此，如果能将原树的信息浓缩，将有用的信息（即应当访问的节点）保留，把大树变成小树，在小树上进行树形DP的即可降低时间复杂度。这就引入了虚树的概念。

概念

对于一棵树 T\mathrm TT，我们可以选取树上的若干个节点作为关键点，构造一棵新的树 T′\mathrm{T}'T′。要求 T′\mathrm T'T′ 包含选取的关键点和所有两个关键点的 LCA\text{LCA}LCA，且 T′\mathrm T'T′ 上的所有节点符合在 T\mathrm TT 中的祖先后代关系。满足上述条件的 T′\mathrm T'T′ 是原树 T\mathrm TT 的虚树（Virtual Tree）。

虚树的构造方法

构造方案简述

设原树为 T\mathrm TT，有 nnn 个节点，编号为 1∼n1\sim n1∼n，默认 111 为根节点；在 T\mathrm TT 上选取 kkk 个关键点，构造的虚树为 T′\mathrm T'T′；还未开始构造虚树时，T′\mathrm T'T′ 为空。kkk 个关键点已经确定，接下来要确定任意两个关键点的 LCA\text{LCA}LCA。若暴力枚举两个关键点的 LCA\text{LCA}LCA，时间复杂度为 O(k2)O(k^2)O(k2)，这显然是不行的，接下来介绍一种利用单调栈构造虚树的方法。
设单调栈 sss，初始时为空。首先明确，对于单调栈 sss 中的元素，其在 T\mathrm TT 中的 DFS\text{DFS}DFS 序是自底向上递增的，其本质上是维护了虚树上的一条树链，sss 中的元素自底向上形成虚树上的链。
理论上，T′\mathrm T'T′ 只需要包含 T\mathrm TT 中应当访问的节点即可；但为了方便起见，默认将根节点 111 添加进 T′\mathrm T'T′。若 111 本应就必须包含在虚树中，那么当然应该将 111 添加进 T′\mathrm T'T′；若 111 是无用的点，在虚树中添加无用的点也不会影响最终结果，树形DP时多一个点也只会对算法时间产生微乎其微的影响。基于以上，构建虚树的第一步就是将 111 添加进 T′\mathrm T'T′。

获得原树 T\mathrm TT 上点的 DFS\text{DFS}DFS 序，用 dfn(x)dfn(x)dfn(x) 表示节点 xxx 的 DFS\text{DFS}DFS 序。
对关键点按照 T\mathrm TT 的 DFS\text{DFS}DFS 序升序排列。我们依次将关键点添加到 T′\mathrm T'T′ 中，并按照 DFS\text{DFS}DFS 序维护单调栈（栈顶元素的 DFS\text{DFS}DFS 序应当是最大的）。同时，还需要考察当前需要添加的关键点和栈顶元素的 LCA\text{LCA}LCA（指的是 T\mathrm TT 中的 LCA\text{LCA}LCA）。
设当前节点为 xxx，栈顶的指针为 toptoptop，那么栈顶元素就为 stops_{top}stop​。
