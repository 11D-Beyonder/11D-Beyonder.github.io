I. Interesting Computer Game ↬\looparrowright↬

题目描述

Apollo is playing an interesting computer game. There are nnn rounds in the game.
At each round, the computer will give Apollo two integers aia_iai​ and bib_ibi​, and Apollo can do exactly one of the following three actions.
Apollo can do nothing.
If integer aia_iai​ has not been selected in all previous rounds, Apollo can select integer aia_iai​.
If integer bib_ibi​ has not been selected in all previous rounds, Apollo can select integer bib_ibi​.
Apollo has cracked the game, and he has known all the candidate numbers of each round before the game starts. Now he wants to know the maximum number of integers he can select with the optimal strategy.
I believe it would be very simple question for you, please help Apollo solve this question.

输入描述

The first line is an integer TTT (1⩽T⩽10)(1 \leqslant T \leqslant 10)(1⩽T⩽10), which is the number of test cases.
Each test case begins with a line containing a positive integer nnn (1⩽n⩽105)(1 \leqslant n \leqslant 10^5)(1⩽n⩽105), indicating the number of rounds in this game.
Then nnn lines follow. The iii-th line contains two integers aia_iai​ and bib_ibi​ (1⩽ai⩽109,1⩽bi⩽109)(1 \leqslant a_i \leqslant 10^9, 1 \leqslant b_i \leqslant 10^9)(1⩽ai​⩽109,1⩽bi​⩽109), indicating that two integers of the iii-th round.

输出描述

For each test case, output one line containing Case #x: yCase\ \#x:\ yCase #x: y, where xxx is the test case number and yyy is the answer.

分析

考虑用网络最大流解决。
aia_iai​ 和 bib_ibi​ 组成一个二元组，每个二元组至多只能选取其中的一个数字；为了体现这种限制，可以设置 nnn 个点，编号为 1∼n1\sim n1∼n，第 iii 个点有两条边分别连向二元组 (ai,bi)(a_i,b_i)(ai​,bi​) 的两个数，容量为 111。每个数字至多被选取一次，将所因此要将所有二元组的数据离散化后，得到 cntcntcnt 个不同的数字，设为 cntcntcnt 个点，编号为 n+1∼n+cntn+1\sim n+cntn+1∼n+cnt；接着，将代表二元组的 nnn 个点向对应的两个数连边。源点向 nnn 个表示二元组的点连边，容量为 111；cntcntcnt 个表示数据的点向 汇点连边，容量为 111。
按照上述方式建图，满足：每次至多选 (ai,bi)(a_i,b_i)(ai​,bi​) 中的一个数字，已选的数字不会再选，网络最大流即为答案。由于边的容量为 111，因此当一条边在增广路上时，这条边的容量就被修改为 000；且按照图的结构，残量网络的层数较少；基于以上，虽然 Dinic\text{Dinic}Dinic 算法的最坏时间复杂度为 O(n2m)O(n^2m)O(n2m)（nnn 为点数，mmm 为边数），但是远远达不到这一上限，在 2s2s2s 的时限内是可以通过的。
