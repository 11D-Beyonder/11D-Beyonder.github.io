A. Road To Zero

You are given two integers xxx and yyy. You can perform two types of operations:
Pay a dollars and increase or decrease any of these integers by 111. For example, if x=0x=0x=0 and y=7y=7y=7 there are four possible outcomes after this operation:x=0,y=6x=0, y=6x=0,y=6;x=0,y=8x=0, y=8x=0,y=8;x=−1,y=7x=−1, y=7x=−1,y=7;x=1,y=7.x=1, y=7.x=1,y=7.
Pay b dollars and increase or decrease both integers by 111. For example, if x=0x=0x=0 and y=7y=7y=7 there are two possible outcomes after this operation: x=−1,y=6x=−1, y=6x=−1,y=6;x=1,y=8.x=1, y=8.x=1,y=8.
Your goal is to make both given integers equal zero simultaneously, i.e. x=y=0x=y=0x=y=0. There are no other requirements. In particular, it is possible to move from x=1x=1x=1, y=0y=0y=0 to x=y=0x=y=0x=y=0.
Calculate the minimum amount of dollars you have to spend on it.

Input

The first line contains one integer ttt (1≤t≤100)(1≤t≤100)(1≤t≤100) — the number of testcases.
The first line of each test case contains two integers xxx and yyy (0≤x,y≤109)(0≤x,y≤10^9)(0≤x,y≤109).
The second line of each test case contains two integers aaa and bbb (1≤a,b≤109)(1≤a,b≤10^9)(1≤a,b≤109).

Output

For each test case print one integer — the minimum amount of dollars you have to spend.

Example

input

2
1 3
391 555
0 0
9 4

output

1337
0

Note

In the first test case you can perform the following sequence of operations: first, second, first. This way you spend 391391391+++555555555+++391391391===133713371337 dollars.
In the second test case both integers are equal to zero initially, so you don’t have to spend money.

Translation

给两个数 x,yx,yx,y。花费 aaa 的代价可以进行操作 111：选择 x,yx,yx,y 其中的一个数将其增加或者减少 111。花费 bbb 的代价可以进行操作 $2 $：将 x,yx,yx,y 都增加或者减少 111。求将 x,yx,yx,y 都置为 000 的最底花费。

Idea

由于 x,y≥0x,y\ge 0x,y≥0 ，所以先进行几次操作 222，直到 min⁡(x,y)\min(x,y)min(x,y) 为 000 ，接着再执行几次操作 111，直到 max⁡(x,y)\max(x,y)max(x,y) 为 000 ，那么就能用最少的操作次数达到目的。但是最高效率并不意味着最低花费，我们可以将操作 222 视为执行了两次操作 111，那么如果两次操作 111 的花费低于操作 222 ，那么执行两次操作 111 才能达到最低花费的目的。也就是说，如果两次操作 111 的花费低于操作 222，那么就用两次操作 111 的代价代替一次操作 222 的代价。

B. Binary Period

Let’s say string sss has period kkk if si=si+ks_i=s_{i+k}si​=si+k​ for all iii from 111 to ∣s∣−k|s|−k∣s∣−k (∣s∣|s|∣s∣ means length of string sss) and kkk is the minimum positive integer with this property.
Some examples of a period: for s=“0101”s=“0101”s=“0101” the period is k=2k=2k=2, for s=“0000”s=“0000”s=“0000” the period is k=1k=1k=1, for s=“010”s=“010”s=“010” the period is k=2k=2k=2, for s=“0011”s=“0011”s=“0011” the period is k=4k=4k=4.
You are given string ttt consisting only of 000’s and 111’s and you need to find such string sss that:
String sss consists only of 000’s and 111’s;
The length of sss doesn’t exceed 2⋅∣t∣2⋅|t|2⋅∣t∣;
String ttt is a subsequence of string sss;
String sss has smallest possible period among all strings that meet conditions 1—31—31—3.
Let us recall that ttt is a subsequence of sss if ttt can be derived from sss by deleting zero or more elements (any) without changing the order of the remaining elements. For example, t=“011”t=“011”t=“011” is a subsequence of s=“10101”s=“10101”s=“10101”.

The first line contains single integer TTT$ (1≤T≤100)$ — the number of test cases.
Next T lines contain test cases — one per line. Each line contains string ttt$ (1≤|t|≤100)$ consisting only of 000’s and 111’s.

Output

Print one string for each test case — string sss you needed to find. If there are multiple solutions print any one of them.

Example

input

output

Note

In the first and second test cases, s=ts=ts=t since it’s already one of the optimal solutions. Answers have periods equal to 111 and 222, respectively.
In the third test case, there are shorter optimal solutions, but it’s okay since we don’t need to minimize the string sss. String sss has period equal to 111.

Translation

给一个 010101 串 ttt，输出一个 010101 串 sss。要求 ∣s∣≤2×∣t∣|s|\le 2\times |t|∣s∣≤2×∣t∣，ttt 是 sss 的子序列，si=si+ks_i=s_{i+k}si​=si+k​ 且要求 kkk 最小。

Idea

显然，如果 $t=1111111\cdots $ 或 t=000000⋯t=000000\cdotst=000000⋯ ，那么 $t $ 本身就符合条件 1−31-31−3，且此时的 k=1k=1k=1 ，是最小的。若 ttt 中出现了 000 和 111，我们可以在 ttt 中添加一些字符得到 sss ，这样做的好处是 ttt 自然会成为 sss 的子序列；我们让 sss 成为 000 和 111 交替的串，即 s=10101010⋯s=10101010\cdotss=10101010⋯ 或 s=01010101⋯s=01010101\cdotss=01010101⋯ ，此时的 sss 必然满足条件，且此时的 k=2k=2k=2 ，是最小的。

D. Multiple Testcases

So you decided to hold a contest on Codeforces. You prepared the problems: statements, solutions, checkers, validators, tests… Suddenly, your coordinator asks you to change all your tests to multiple testcases in the easiest problem!
Initially, each test in that problem is just an array. The maximum size of an array is kkk. For simplicity, the contents of arrays don’t matter. You have nnn tests — the i-th test is an array of size mim_imi​ (1≤mi≤k)(1≤m_i≤k)(1≤mi​≤k).
Your coordinator asks you to distribute all of your arrays into multiple testcases. Each testcase can include multiple arrays. However, each testcase should include no more than c1c_1c1​ arrays of size greater than or equal to 1(≥1)1 (≥1)1(≥1), no more than c2c_2c2​ arrays of size greater than or equal to 222, …, no more than ckc_kck​ arrays of size greater than or equal to kkk. So now your goal is to create the new testcases in such a way that:

  • each of the initial arrays appears in exactly one testcase;
  • for each testcase the given conditions hold;
  • the number of testcases is minimum possible.
    Print the minimum possible number of testcases you can achieve and the sizes of arrays included in each testcase.

Input

The first line contains two integers nnn and kkk (1≤n,k≤2⋅105)(1≤n,k≤2⋅10^5)(1≤n,k≤2⋅105) — the number of initial tests and the limit for the size of each array.

Each of the next ans lines should contain the description of a testcase in the following format:
ttt a1,a2,…,ata_1, a_2, …, a_ta1​,a2​,…,at​ (1≤t≤n)(1≤t≤n)(1≤t≤n) — the testcase includes ttt arrays, aia_iai​ is the size of the iii-th array in that testcase.

Note

In the first example there is no way to distribute the tests into less than 333 testcases. The given answer satisfies the conditions: each of the testcases includes no more than 444 arrays of size greater than or equal to 111 and no more than 111 array of sizes greater than or equal to 222 and 333.
Note that there are multiple valid answers for this test. For example, testcases with sizes [[2],[1,2],[3]][[2],[1,2],[3]][[2],[1,2],[3]] would also be correct.

E. Placing Rooks

Calculate the number of ways to place n rooks on n×n chessboard so that both following conditions are met:
- each empty cell is under attack;
- exactly kkk pairs of rooks attack each other.
An empty cell is under attack if there is at least one rook in the same row or at least one rook in the same column. Two rooks attack each other if they share the same row or column, and there are no other rooks between them. For example, there are only two pairs of rooks that attack each other in the following picture:

One of the ways to place the rooks for n=3n=3n=3 and k=2k=2k=2
Two ways to place the rooks are considered different if there exists at least one cell which is empty in one of the ways but contains a rook in another way.
The answer might be large, so print it modulo 998244353998244353998244353.
