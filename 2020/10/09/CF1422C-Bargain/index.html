CF1422C Bargain

发表于2020-10-09|更新于2023-01-31|题解
|字数总计:869|阅读时长:4分钟|阅读量:

Sometimes it is not easy to come to an agreement in a bargain. Right now Sasha and Vova can’t come to an agreement: Sasha names a price as high as possible, then Vova wants to remove as many digits from the price as possible. In more details, Sasha names some integer price nnn, Vova removes a non-empty substring of (consecutive) digits from the price, the remaining digits close the gap, and the resulting integer is the price.
For example, is Sasha names 121312112131211213121, Vova can remove the substring 131213121312, and the result is 121121121.
It is allowed for result to contain leading zeros. If Vova removes all digits, the price is considered to be 000.
Sasha wants to come up with some constraints so that Vova can’t just remove all digits, but he needs some arguments supporting the constraints. To start with, he wants to compute the sum of all possible resulting prices after Vova’s move.
Help Sasha to compute this sum. Since the answer can be very large, print it modulo 109+710^9+7109+7.

Input

The first and only line contains a single integer nnn (1≤n<10105)(1≤n<10^{10^5})(1≤n<10105).

Output

In the only line print the required sum modulo 109+710^9+7109+7.

Note

Consider the first example.
Vova can choose to remove 1,0,7,10,07,or1071, 0, 7, 10, 07, or 1071,0,7,10,07,or107. The results are 07,17,10,7,1,007, 17, 10, 7, 1, 007,17,10,7,1,0. Their sum is 424242.

Translation

给一个正整数 nnn，可以选择 nnn 上连续的位将之取出，于是 nnn 就成了新的数。求所有操作剩下的数的和。

Idea

如果去掉的连续位在 nnn 的头部或 nnn 的尾部，可 O(n)O(n)O(n) 统计贡献。
