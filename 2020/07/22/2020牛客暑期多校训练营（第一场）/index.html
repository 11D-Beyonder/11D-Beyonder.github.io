H. Minimum-cost Flow ↬\looparrowright↬

题目描述

Bobo has a network of nnn nodes and mmm arcs. The iii-th arc goes from the aia_iai​-th node to the bib_ibi​-th node, with cost cic_ici​.
Bobo also asks qqq questions. The iii-th question is specified by two integers uiu_iui​ and viv_ivi​, which is to ask the minimum cost to send one unit of flow from the 111-st node to the nnn-th node, when all the edges have capacity uivi\frac{u_i}{v_i}vi​ui​​ (a fraction).

分析

运行 qqq 次最小费用流算法显然是不现实的，应当考虑的是：运行一次最小费用流，进行一些预处理，使得每次询问时能够在较短时间内给出答案。
题中有一重要信息：每条边的容量都相同。设这个容量为 capcapcap 。利用 MCMF\text{MCMF}MCMF 算法求最小费用最大流，会不断用 SPFA\text{SPFA}SPFA 算法寻找单位费用和最小的增广路；由于每条边的容量都为 capcapcap，故每条边只会存在满流或零流的情况，每次增广获得的最大流改进量恒为 capcapcap 。若源点流量为 flowflowflow 且 flowflowflow 不超过最大流，就是最小费用流模型。对于这样特殊的（每条边容量都为 capcapcap）最小费用流模型，仍然可以利用 MCMF\text{MCMF}MCMF 算法，每次增广后相当于将流量 flowflowflow 减少 capcapcap，产生的费用是增广路单位费用和与 capcapcap 的乘积；而最后一次增广时费用的改进量为剩余流量 flow mod capflow\bmod capflowmodcap 与增广路单位费用和的乘积。进行多轮增广，直至剩余流量为零，就能得到最小费用流。
由于每条边的费用 cic_ici​ 是不变的，因此不论 capcapcap 如何变化，只要源点流量不超过当前网络的最大流，对 qqq 次询问中的每一张网络进行 MCMF\text{MCMF}MCMF 算法，找到的增广路都是相同的，且每条增广路的顺序也保持不变。因此，只需要进行一次最小费用流的计算，记录下每一条增广路的单位费用即可，根据 MCMF\text{MCMF}MCMF 算法原理，这些增广路是按照单位费用和由小到大获得的，排序操作是多余的。
不妨令每条边的容量为 111，求一次最小费用最大流，记录下每一条增广路的单位费用。对于每一次询问，源点流量为 111，每条边容量为 uv\frac{u}{v}vu​。为了减少计算的误差，可以将所有数量扩大 vvv 倍，即网络中源点流量为 vvv，每条边容量为 uuu。若源点流量 vvv 不超过网络最大流，那么求固定流量 vvv 的最小费用流得过程中，增广路为最小费用最大流得增广路子集，且每一轮 SPFA\text{SPFA}SPFA 获得的增广路一定是当前所有存在的增广路中费用最小的。每一轮增广，源点流量就减少 uuu，相应的费用为该增广路的单位费用与 uuu 的乘积；最后一次增广，源点流量减少 u mod vu\bmod vumodv，对应费用也要相应改变；最终，源点流量为 000，求得最小费用流 ans\mathrm{ans}ans。ansv\frac{\mathrm{ans}}{v}vans​ 即为一次询问的答案。
