A. Sign Flipping ↬\looparrowright↬

You are given nnn integers a1,a2,⋯ ,ana_1,a_2,\cdots,a_na1​,a2​,⋯,an​, where nnn is odd. You are allowed to flip the sign of some (possibly all or none) of them. You wish to perform these flips in such a way that the following conditions hold:

  1. At least n−12\frac{n−1}{2}2n−1​ of the adjacent differences ai+1−aia_{i+1}−a_iai+1​−ai​ for i=1,2,⋯ ,n−1i=1,2,\cdots,n−1i=1,2,⋯,n−1 are greater than or equal to 000.
  2. At least n−12\frac{n−1}{2}2n−1​ of the adjacent differences ai+1−aia_{i+1}−a_iai+1​−ai​ for i=1,2,⋯ ,n−1i=1,2,\cdots,n−1i=1,2,⋯,n−1 are less than or equal to 000.
    Find any valid way to flip the signs. It can be shown that under the given constraints, there always exists at least one choice of signs to flip that satisfies the required condition. If there are several solutions, you can find any of them.

Input

The input consists of multiple test cases. The first line contains an integer ttt(1≤t≤500)(1≤t≤500)(1≤t≤500) — the number of test cases. The description of the test cases follows.
The first line of each test case contains an integer nnn (3≤n≤993≤n≤993≤n≤99, nnn is odd) — the number of integers given to you.
The second line of each test case contains nnn integers a1,a2,⋯ ,ana_1,a_2,\cdots,a_na1​,a2​,⋯,an​ (−109≤ai≤109)(−10^9≤a_i≤10^9)(−109≤ai​≤109) — the numbers themselves.

Output

B. Neighbor Grid ↬\looparrowright↬

You are given a grid with nnn rows and mmm columns, where each cell has a non-negative integer written on it. We say the grid is good if for each cell the following condition holds: if it has a number k>0k>0k>0 written on it, then exactly kkk of its neighboring cells have a number greater than 000 written on them. Note that if the number in the cell is 000, there is no such restriction on neighboring cells.
You are allowed to take any number in the grid and increase it by 111. You may apply this operation as many times as you want, to any numbers you want. Perform some operations (possibly zero) to make the grid good, or say that it is impossible. If there are multiple possible answers, you may find any of them.
Two cells are considered to be neighboring if they have a common edge.

Input

The input consists of multiple test cases. The first line contains an integer ttt (1≤t≤5000)(1≤t≤5000)(1≤t≤5000) — the number of test cases. The description of the test cases follows.
The first line of each test case contains two integers nnn and mmm (2≤n,m≤300)(2≤n,m≤300)(2≤n,m≤300) — the number of rows and columns, respectively.
The following nnn lines contain mmm integers each, the jjj-th element in the iii-th line ai,ja_{i,j}ai,j​ is the number written in the jjj-th cell of the iii-th row (0≤ai,j≤109)(0≤a_{i,j}≤10^9)(0≤ai,j​≤109).
It is guaranteed that the sum of n×mn\times mn×m over all test cases does not exceed 10510^5105.

Otherwise, print a single line containing “YES”, followed by nnn lines each containing mmm integers, which describe the final state of the grid. This final grid should be obtainable from the initial one by applying some operations (possibly zero).

In one operation, you can choose an index iii (1≤i<n)(1≤i<n)(1≤i<n) such that ai<ai+1a_i<a_{i+1}ai​<ai+1​, and remove either aia_iai​ or ai+1a_{i+1}ai+1​ from the array (after the removal, the remaining parts are concatenated).

D. Replace by MEX ↬\looparrowright↬

You’re given an array of nnn integers between 000 and nnn inclusive.
In one operation, you can choose any element of the array and replace it by the MEX of the elements of the array (which may change after the operation).
For example, if the current array is [0,2,2,1,4][0,2,2,1,4][0,2,2,1,4], you can choose the second element and replace it by the MEX of the present elements — 333. Array will become [0,3,2,1,4][0,3,2,1,4][0,3,2,1,4].
You must make the array non-decreasing, using at most 2n2n2n operations.
It can be proven that it is always possible. Please note that you do not have to minimize the number of operations. If there are many solutions, you can print any of them.
An array b[1⋯n]b[1\cdots n]b[1⋯n] is non-decreasing if and only if b1≤b2≤⋯≤bnb_1≤b_2≤\cdots≤b_nb1​≤b2​≤⋯≤bn​.
The MEX (minimum excluded) of an array is the smallest non-negative integer that does not belong to the array. For instance:

  • The MEX of [2,2,1][2,2,1][2,2,1] is 000, because 000 does not belong to the array.
  • The MEX of [3,1,0,1][3,1,0,1][3,1,0,1] is 222, because 000 and 111 belong to the array, but 222 does not.
  • The MEX of [0,3,1,2][0,3,1,2][0,3,1,2] is 444 because 0,1,20, 1, 20,1,2 and 333 belong to the array, but 444 does not.

It’s worth mentioning that the MEX of an array of length nnn is always between 000 and nnn inclusive.

Input

The first line contains a single integer ttt (1≤t≤200)(1≤t≤200)(1≤t≤200) — the number of test cases. The description of the test cases follows.

It is guaranteed that the sum of nnn over all test cases doesn’t exceed 100010001000.

Output

For each test case, you must output two lines:
The first line must contain a single integer kkk (0≤k≤2n)(0≤k≤2n)(0≤k≤2n) — the number of operations you perform.
The second line must contain kkk integers x1,⋯ ,xkx_1,\cdots,x_kx1​,⋯,xk​ (1≤xi≤n)(1≤x_i≤n)(1≤xi​≤n), where xix_ixi​ is the index chosen for the iii-th operation.

E. Inversion Swap Sort ↬\looparrowright↬

Madeline has an array aaa of nnn integers. A pair (u,v)(u,v)(u,v) of integers forms an inversion in aaa if: 1≤u<v≤n1≤u<v≤n1≤u<v≤n, au>ava_u>a_vau​>av​.
Madeline recently found a magical paper, which allows her to write two indices uuu and vvv and swap the values aua_uau​ and ava_vav​. Being bored, she decided to write a list of pairs (ui,vi)(u_i,v_i)(ui​,vi​) with the following conditions: all the pairs in the list are distinct and form an inversion in aaa, all the pairs that form an inversion in aaa are in the list.
Starting from the given array, if you swap the values at indices u1u_1u1​ and v1v_1v1​, then the values at indices u2u_2u2​ and v2v_2v2​ and so on, then after all pairs are processed, the array aaa will be sorted in non-decreasing order.

Idea

首先考虑 aaa 是 1∼n1\sim n1∼n 全排列时的做法。
设 posxpos_xposx​ 表示 xxx 这个数在 aaa 中出现的位置。
从边界入手，先尝试把 nnn 放到排列的最后一个位置，然后转化为规模减一的子问题。假设操作后，我们得到的排列为 bbb，要求 bbb 满足：bn=nb_n=nbn​=n；∀ 1≤i,j<n∀\ 1≤i,j<n∀ 1≤i,j<n，若 ai<aja_i<a_jai​<aj​，则 bi<bjb_i<b_jbi​<bj​；∀1≤i,j<n∀ 1≤i,j<n∀1≤i,j<n，若 ai>aja_i>a_jai​>aj​，则 bi>bjb_i>b_jbi​>bj​。也就是说，要保证前面的数的相对大小关系不变，这样才能转化为一个等价的子问题。我们可以依次交换索引对应的值：(posan+1,n)(pos_{a_n+1},n)(posan​+1​,n),,,(posan+2,n)(pos_{a_n+2},n)(posan​+2​,n),,,(posan+3,n)(pos_{a_n+3},n)(posan​+3​,n),,,⋯\cdots⋯,,,(posn,n)(pos_n,n)(posn​,n)。举例，操作 [1,5,3,4,2][1,5,3,4,2][1,5,3,4,2] 的过程：[1,5,3,4,2][1,5,3,4,2][1,5,3,4,2]→\rightarrow→[1,5,2,4,3][1,5,2,4,3][1,5,2,4,3]→\rightarrow→[1,5,2,3,4][1,5,2,3,4][1,5,2,3,4]→\rightarrow→[1,4,2,3,5][1,4,2,3,5][1,4,2,3,5]。容易发现，这样一轮操作完成后，nnn 被放到了最后；同时，相当于前面所有大于 ana_nan​的数，全部减去 111；因此，显然前面所有数的相对大小关系不变，剩下的是一个规模更小的子问题。接着以索引 n−1n-1n−1 的位置为末尾，将数值 n−1n-1n−1 放到最后。不断重复，即可得到一个标准排列。
基于以上，当 aaa 是一个全排列时，一定有解。
现在考虑 aaa 不是全排列的时候。事实上，可以设置数值为第一关键字，位置为第二关键字，强行转成一个全排列。举例，a=[1,4,3,5,3,1]a=[1,4,3,5,3,1]a=[1,4,3,5,3,1]，重新赋值转换为全排列后 a=[1,5,3,6,4,2]a=[1,5,3,6,4,2]a=[1,5,3,6,4,2]。发现转成全排列后，序列里的逆序对个数和原来是一样的。可以进行略证：a,a,ca,a,ca,a,c 中(1,3)(1,3)(1,3) 是逆序对，重新赋值后为 2,3,12,3,12,3,1，由于对于，相同数字，位置靠后的重新赋值后数值也更大，不会产生逆序对，所以逆序对个数是和原来一样的。
综上所述，对于任意的数组 aaa，将 aaa 经过转换后直接按全排列求解即可。对于一个全排列，总的交换次数至多为 (n−1)+(n−2)+⋯+1=n(n−1)2(n-1)+(n-2)+\cdots+1=\frac{n(n-1)}{2}(n−1)+(n−2)+⋯+1=2n(n−1)​，满足题目限制条件。
